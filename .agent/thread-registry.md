# Thread Registry

> Master index of all discussion/planning threads across the project.
> Agents scan this to find relevant context. Maintainer keeps it current.

## How to Read This

| Column   | Meaning |
|----------|---------|
| ID       | `<feature>/<slug>` — matches the thread-id in frontmatter |
| Status   | `active` · `blocked` · `stale` · `good-enough` · `closed` |
| Progress | `N/M` from frontmatter — steps done / total steps |
| Parent   | Thread ID of parent, or `root` |
| Path     | Relative path to the `.plans~/` document |
| Summary  | One-line scan description |
